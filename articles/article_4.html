GitHub Copilot Instructions Files 完整指南

原文來源：Medium - Max Yermakhanov
以下為核心要點摘要，附英文技術詞彙對照（紫色）幫助理解。

核心摘要

GitHub Copilot 提供多種 instructions files（指令檔案），讓開發者能透過 Markdown 檔案客製化 AI 助理的行為和回應。這些檔案為提供 context（上下文） 和 instructions（指令） 建立了專門且可預測的位置，能夠針對不同範圍（組織、專案、資料夾）調整 Copilot 的回應方式。

主要檔案類型

1. copilot-instructions.md

copilot-instructions.md 包含適用於所有 GitHub Copilot requests（請求） 的指令和偏好設定，不僅限於 AI agents（AI 代理）。

作用範圍：

  • Organization-wide（組織層級）：透過組織設定配置，適用於組織成員的所有 Copilot 請求
  • Repository-wide（專案層級）：將檔案放在 .github/copilot-instructions.md，適用於該專案內的所有請求
  • Path-specific（路徑特定）：使用 NAME.copilot-instructions.md 檔案存放在 .github/instructions/ 資料夾中，可透過 applyTo 和 excludeAgent 關鍵字指定適用範圍

範例應用：

  • 建立 backend.copilot-instructions.md，包含後端開發特定指令
  • 設定 applyTo: "**/*.ts,**/*.tsx" 讓指令只適用於 TypeScript 檔案
  • 當 path-specific instructions（路徑特定指令） 生效時，會同時使用 repository-wide（專案層級） 和 path-specific 指令

當 Copilot Chat 使用 repository custom instructions（專案自訂指令） 時，指令檔案會作為 reference（參考資料） 加入回應中。你可以在 Chat panel（對話面板） 頂部展開 references 清單，查看是否使用了 .github/copilot-instructions.md。

2. AGENTS.md

可在專案內任意位置建立一或多個 AGENTS.md 檔案。當 Copilot 工作時，directory tree（目錄樹） 中最接近的 AGENTS.md 檔案會優先於其他 agent instructions files（代理指令檔案）。

你也可以使用存放在專案根目錄的單一 CLAUDE.md 或 GEMINI.md 檔案作為替代方案。

詳細文件可參考：agents.md

3. *.agent.md

Custom agents（自訂代理） 讓 GitHub Copilot 能透過簡單的 file-based configuration（檔案式配置） 進行「專業化」調整，使用者和組織能輕鬆特製自己的 Copilot coding agent（編碼代理）。

4. prompts/*.md

Prompt files（提示詞檔案）（通常稱為 slash commands（斜線指令））讓你能將常用提示詞封裝為獨立的 Markdown 檔案，可直接從對話中觸發。每個檔案包含 task-specific context（任務特定上下文） 以及執行任務的指令。

與 custom instructions 的差異：

  • Custom instructions 會自動套用
  • Prompt files 需在 chat session（對話會話） 中手動呼叫

適用場景：

  • 標準化頻繁任務，如建立新元件、執行和修復測試、準備 pull requests（拉取請求）
  • 調整 custom agent（自訂代理） 的行為，例如生成簡潔的 implementation plan（實作計畫） 或建立 mock API call mockups（模擬 API 呼叫原型）

Prompt files 存放在 repository 的 .github/prompts 目錄中，每個檔案應以 .md 副檔名命名。

5. SKILLS.md

Agent Skills（代理技能） 是自包含的目錄，將 instructions（指令） 和 supporting resources（支援資源） 打包在一起，用於擴展 AI agent 針對專門任務的能力。

Skills 的獨特之處：

  • 能夠打包 supporting assets（支援素材），如 scripts（腳本）、code samples（程式碼範例） 或 reference data（參考資料）
  • agents 可在執行任務時運用這些資源
  • 適合需要更多 context（上下文） 或資源的 complex, multi-step tasks（複雜多步驟任務）

遵循 Agent Skills specification，每個技能都包含一個 SKILL.md 檔案，內含 agents 按需載入的詳細指引。透過將相關指令和素材組織在一起，Skills 讓 agents 能以更強大且高效的方式執行專門功能。

如何選擇

  • Prompt files：用於輕量、單一任務的提示詞
  • Custom agents：當你需要具備 persistent persona（持續性人格）、自有 tool restrictions（工具限制） 和 handoffs（交接） 的代理時使用
  • Agent skills：當你需要具備 scripts（腳本） 和 resources（資源） 的 portable, multi-file capability（可攜式多檔案能力） 時使用

補充資源

  • GitHub Copilot Plugins：雖非 Markdown 檔案，但是透過整合外部 services（服務） 和 APIs 來擴展 Copilot 功能的強大方式
  • GitHub Copilot Hooks：在 Copilot 中回應特定 events（事件） 或 conditions（條件） 觸發 custom actions（自訂動作） 或 workflows（工作流程）
  • GitHub Agentic Workflows：協調多個 agents、skills 和 prompts 共同完成需要 coordination（協調） 和 handoffs（交接） 的複雜任務

對程式設計師的意義

這篇文章完整解析了 GitHub Copilot 的各種客製化機制，幫助開發者：

  • 提升效率：透過 prompt files 標準化常見任務，減少重複輸入
  • 精準控制：使用不同層級的 instructions 檔案，針對組織、專案、資料夾設定合適的 AI 行為
  • 團隊協作：將 coding guidelines（編碼準則）、best practices（最佳實踐） 編碼為 instructions 檔案，確保團隊一致性
  • 進階應用：結合 Skills、Agents、Hooks 建構複雜的 AI-powered workflows（AI 驅動工作流程）

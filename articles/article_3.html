mcp2cli - 節省 90% Token 使用量的 CLI 工具

原文來源：Reddit - r/Python
以下為核心要點摘要，附英文技術詞彙對照（紫色）幫助理解。

核心摘要

mcp2cli 是一款 runtime CLI generator（運行時 CLI 生成器），解決 Large Language Models（大型語言模型） 連接 MCP servers（MCP 伺服器） 或 OpenAPI servers（OpenAPI 伺服器） 時的 token（令牌） 浪費問題。

傳統方式在每次對話中都會將所有 tool schemas（工具模式定義） 注入到 system prompt（系統提示詞） 中，導致即使不使用這些工具，也會消耗大量 tokens。例如，6 個 MCP servers 包含 84 個 tools，在對話開始前就會燒掉約 15,500 個 tokens。

核心功能

  • 運行時生成：無需 codegen（程式碼生成） 或 compilation（編譯），直接從 MCP server URL 或 OpenAPI spec（規格） 生成完整功能的 CLI
  • 按需發現：LLM 可透過 --list 和 --help 指令發現並呼叫工具，而非在每次對話中注入完整 JSON schemas（JSON 模式定義）
  • 大幅節省 tokens：將 15,500 tokens 的 system prompt 縮減為 67 tokens，並在需要時才載入工具細節，整體對話中可節省 92–99% 的 token 使用量
  • 支援雙協定：同時支援 MCP 和 OpenAPI 協定，使用相同介面
  • OAuth 支援：內建 authorization code + PKCE（帶證明密鑰的授權碼） 和 client credentials flows（客戶端憑證流程），自動處理 token caching（令牌快取） 和 refresh（更新）

使用範例

安裝

pip install mcp2cli

連接 MCP Server

# 列出可用工具
mcp2cli --mcp https://mcp.example.com/sse --list

# 呼叫搜尋工具
mcp2cli --mcp https://mcp.example.com/sse search --query "test"

連接 OpenAPI Spec

# 列出 API 端點
mcp2cli --spec https://petstore3.swagger.io/api/v3/openapi.json --list

# 建立寵物資料
mcp2cli --spec ./openapi.json create-pet --name "Fido" --tag "dog"

使用 stdio 的 MCP Server

mcp2cli --mcp-stdio "npx @modelcontextprotocol/server-filesystem /tmp" \
  read-file --path /tmp/hello.txt

效能比較

vs. Native MCP Tool Injection

原生 MCP 在每次對話中注入完整 JSON schemas（每個工具約 121 tokens）。以 30 個工具進行 15 輪對話為例，光是 schemas 就消耗約 54,500 tokens。mcp2cli 只需約 2,300 tokens（減少 96%），因為只在 LLM 實際需要時才載入工具細節。

vs. Anthropic's Tool Search

Tool Search 是 Anthropic 專屬的 API feature（功能），透過 search index（搜尋索引） 延遲載入工具（約 500 tokens）。mcp2cli 則：

  • Provider-agnostic（供應商無關）：適用於任何能執行 shell commands（命令列指令） 的 LLM
  • 更精簡：--list 每個工具約 16 tokens，遠低於 Tool Search 載入的 121 tokens

vs. Codegen Tools (如 openapi-generator)

傳統 codegen tools 會產生 static client code（靜態客戶端程式碼），當 spec 改變時需要重新生成。mcp2cli 無需 codegen，在 runtime（執行時期） 直接讀取 spec。雖然它是 generic CLI（通用命令列介面） 而非 typed SDK（型別化軟體開發套件），但這正是 LLM tool use（工具使用） 的最佳選擇。

對程式設計師的意義

如果你正在開發 LLM-powered agents（大型語言模型驅動的代理） 或需要連接外部 APIs 的工作流程，這個工具可以大幅降低 context window（上下文視窗） 被 tool schemas 塞滿的問題。

適用場景：

  • 連接 Claude、GPT、Gemini 或 local models（本地模型） 到 MCP servers 或 REST APIs
  • 需要快速為任何 OpenAPI 或 MCP endpoint（端點） 建立 CLI，無需撰寫 client code（客戶端程式碼）
  • 在 production environments（正式環境） 中建構成本敏感的 AI 工作流程

關鍵優勢：

  • 零等待時間：指向 URL 即可立即使用，spec 更新後下次呼叫自動生效
  • Spec caching（規格快取）：抓取的 specs 會在本地快取，並有可配置的 TTL（存活時間）
  • 安全處理：支援 env: 和 file: 前綴處理敏感值，避免出現在 process listings（程序清單） 中

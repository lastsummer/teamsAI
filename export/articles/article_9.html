摘要

作者 Antonio Di Motta 使用官方 .NET SDK 1.0 建立符合 MCP 規範的伺服器，後端使用 PostgreSQL，示範如何讓 LLM 以自然語言查詢企業資料。

技術架構

[McpServerTool] — 工具定義

使用自訂屬性標記函式，讓 LLM 能識別並呼叫。例如 GetCustomer 工具可以非同步查詢資料庫，依姓名篩選客戶資料。

[McpServerPrompt] — 提示模板

使用 McpServerPrompt 屬性建立複雜的上下文回應模板，如 SuggestProductByCustomer，包含 system、user 與 assistant 角色的結構化訊息。

自然語言查詢範例

支援如下的自然語言請求：

  • 「給我客戶清單」
  • 「告訴我最昂貴的訂單」
  • 「USB-C Hub 總共訂購了幾個？」
程式碼片段概覽
// Tool 定義範例 [McpServerTool, Description("Get customer by name")] public static async Task<Customer> GetCustomer( [Description("Customer full name")] string name, AppDbContext db) { return await db.Customers .FirstOrDefaultAsync(c => c.FullName == name); }
重點結語

MCP 已支援 .NET 官方 SDK 並達到生產就緒狀態，讓 .NET 開發者能夠輕鬆將後端系統暴露給 AI 助手。完整範例已在 GitHub 上開源。

Summary

Antonio Di Motta builds an MCP-compliant server using the official .NET SDK 1.0 with a PostgreSQL backend, demonstrating how LLMs can query enterprise data using natural language.

Technical Architecture

[McpServerTool] — Tool Definitions

Custom attributes mark functions that LLMs can recognize and execute. The GetCustomer tool demonstrates async database querying with name-based filtering.

[McpServerPrompt] — Prompt Templates

Use McpServerPrompt attribute to create complex contextual response templates like SuggestProductByCustomer, with structured system, user, and assistant role messages.

Natural Language Query Examples

The server responds to natural language queries such as:

  • "Give me the customer list"
  • "Tell me the most expensive order"
  • "How many USB-C Hubs have been ordered?"

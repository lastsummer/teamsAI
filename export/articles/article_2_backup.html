摘要

Builder.io 的 Vishwas Gopinath 整理了 Claude Code CLI 工具的 8 項快速自訂設定，強調大多數調整只需幾分鐘即可完成，甚至可以直接請 Claude 自己完成設定。

8 項核心自訂
設定 01

建立 Shell 別名 (cc alias)

在 ~/.zshrc 或 ~/.bashrc 中加入 alias cc='claude --dangerously-skip-permissions'，略過每次的確認提示。額外可加 ccp（計畫模式）與 ccr（恢復上次會話）別名。

設定 02

Context Window 自動壓縮閾值

預設在 95% 容量才觸發壓縮，建議調整至 60–75%，透過環境變數 CLAUDE_AUTOCOMPACT_PCT_OVERRIDE 設定，讓對話保持連貫。

設定 03

個人 CLAUDE.md 全域設定

在 ~/.claude/CLAUDE.md 建立全域個人設定，跨所有會話保持偏好。建議保持在 50 行以內，以具體規則（如偏好 pnpm 而非 npm）取代模糊指示。

設定 04

即時狀態列顯示

在終端底部顯示即時資訊：當前目錄、git 分支狀態，以及以色彩編碼的 context 使用量（50% 以下綠色、50–80% 黃色、80% 以上紅色）。

設定 05

編輯後自動格式化

設定 PostToolUse hook，在每次檔案編輯後自動執行 Prettier，還可串接 ESLint 進行整合式程式碼品質維護。

設定 06

自訂 Spinner 動詞

透過設定替換預設的等待動畫文字，改為個人化詞彙，或在預設清單後附加自訂動詞。

設定 07

完成時播放音效

設定任務完成後自動播放通知音效，讓你在等待長時間任務時可以切換做其他事。

設定 08

設定輸出風格

自訂 Claude 回應的格式與風格，確保輸出符合你的開發偏好與閱讀習慣。

重點結語

文章強調 Claude Code 本身就能協助完成大多數設定——只需描述你想要的偏好，Claude 就會處理技術細節，大幅降低手動編輯設定檔的摩擦力。

Summary

Vishwas Gopinath from Builder.io presents 8 practical customizations for Claude Code, Anthropic's CLI tool. Most adjustments require minimal setup time and can be implemented by simply asking Claude what you want.

8 Key Customizations
Setting 01

Shell Alias Setup

Add alias cc='claude --dangerously-skip-permissions' to your shell config to skip permission prompts. Also add ccp for plan mode and ccr to resume sessions.

Setting 02

Context Window Auto-Compaction

Rather than waiting until 95% capacity, trigger compaction at 60-75% via the CLAUDE_AUTOCOMPACT_PCT_OVERRIDE environment variable to maintain conversation coherence.

Setting 03

Personal CLAUDE.md File

Create ~/.claude/CLAUDE.md for persistent preferences across all sessions. Keep under 50 lines with specific rules (e.g. prefer pnpm over npm) rather than vague guidance.

Setting 04

Live Status Line

Display a terminal status line showing current directory, git branch, and color-coded context usage (green <50%, yellow 50–80%, red >80%).

Setting 05

Automatic Code Formatting

Configure PostToolUse hooks to auto-run Prettier after each file edit. Chain ESLint for integrated code quality maintenance.

Setting 06

Custom Spinner Verbs

Replace default spinner messages with personalized alternatives, or append custom verbs to the defaults list.

Setting 07

Completion Sound

Configure a sound notification when tasks complete, allowing you to multitask while waiting for long-running operations.

Setting 08

Output Style

Customize Claude's response format and style to match your development preferences and reading habits.

Key Takeaway

Claude Code itself can configure most settings — just describe your preferences and Claude handles the technical setup, reducing friction compared to manual config file editing.

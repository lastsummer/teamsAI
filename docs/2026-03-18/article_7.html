OWASP MCP 前十大漏洞：為何您的 AI Agents 容易受到攻擊

原文來源：dev.to
以下為核心要點摘要，附英文技術詞彙對照（紫色）幫助理解。

核心摘要

OWASP Foundation（開放網路應用安全專案基金會）發布了 MCP Top 10 安全漏洞框架，涵蓋 Model Context Protocol（模型上下文協議）部署中最常見的十類安全風險。作者同時開發了 Ferrok（MCP 安全掃描器）以協助自動偵測這些漏洞。

十大漏洞速覽

  • MCP-01 Tool Poisoning（工具投毒）：攻擊者在 tool descriptions（工具描述）中注入惡意指令，操縱 AI 執行非預期動作（例如：讀取 ~/.ssh/id_rsa）
  • MCP-02 Excessive Permissions（過度權限）：工具缺乏 least-privilege constraints（最小權限限制），授予過寬的檔案系統或網路存取
  • MCP-03 Schema Misuse（Schema 誤用）：弱型別輸入驗證，類似 SQL injection（SQL 注入）的注入漏洞
  • MCP-04 Transport Security（傳輸安全）：未加密連線、設定中的 hardcoded credentials（硬編碼憑證）
  • MCP-05 Insufficient Access Controls（存取控制不足）：缺乏認證、rate limiting（速率限制）或授權檢查
  • MCP-06 Data Leakage（資料洩漏）：工具回應暴露內部路徑、資料庫結構或 stack traces（堆疊追蹤）
  • MCP-07 Insecure Data Handling（不安全資料處理）：PII（個人識別資訊）或憑證出現在 chat logs（對話紀錄）或訓練資料中
  • MCP-08 Lack of Logging（缺乏日誌）：大多數伺服器沒有 audit trails（稽核軌跡）
  • MCP-09 Third-Party Server Risks（第三方伺服器風險）：透過 npx -y 安裝未審查套件的 supply chain vulnerabilities（供應鏈漏洞）
  • MCP-10 Authorization Protocol Violations（授權協議違規）：OAuth 實作不當、token 儲存不安全

對程式設計師的意義

MCP 普及速度遠超安全成熟度。現在就應審計 tool descriptions、限制 input schemas、強制 HTTPS、檢查第三方套件供應鏈，並為所有 MCP tool 呼叫建立 comprehensive logging（完整日誌）。凡是在生產環境部署 AI agent 的團隊，這份清單是必讀的安全基準。

從 1 到 3 個平行 Claude Code Agents：突破 16% 使用率的方法

原文來源：dev.to
以下為核心要點摘要，附英文技術詞彙對照（紫色）幫助理解。

核心摘要

作者在 Claude Code Max 20x 方案（$200/月）上只達到 16% 使用率，透過逐步擴展至 3 個平行 agents（並行代理），最終突破 45%+ 使用率並頻繁觸及每週速率限制。文章的核心論點是：先讓第一個 agent 真正忙起來，再考慮增加更多。

關鍵技術：Git Worktrees

多個 agents 在同一個 repository（代碼倉庫）工作會產生 merge conflicts（合併衝突）和資源競爭。解決方案是 Git worktrees（Git 工作樹）——為同一個 repository 建立多個獨立的工作目錄，各自連結到不同的 branch（分支），共享 git history（提交歷史）。

實作步驟

  • 第一步：確認 agent 真的很忙。若 agent 在等待你的回應或頻繁被中斷，優先解決這些問題，而非增加 agent 數量
  • 第二步：任務隔離。前後端分離、基礎設施 vs 應用代碼分離、或 feature pipeline（功能流水線）等方式建立真正的工作邊界
  • 第三步：基礎設施並行化。各 agent 需要獨立的 database（資料庫）、不同的 port（連接埠）和環境配置
  • 第四步：Claude Skills 自動化。作者開發了 Skills 腳本自動完成 worktree 建立、port 分配和資料庫配置，已在 GitHub 開源
💡 核心建議：「不要從工具開始。先讓第一個 agent 完全忙碌，然後再為第二個找一個隔離的任務。」

對程式設計師的意義

Claude Code Max 方案的使用率低落是普遍問題。Git worktrees 是解決 parallel agent（並行代理）衝突最乾淨的方案，且不需要複雜的基礎設施改變。這個模式適用於任何需要 multi-agent（多代理）開發的團隊，特別是在大型 codebase（代碼庫）重構或多功能同步開發時。

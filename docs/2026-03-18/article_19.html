[GHSA-2cpp-j2fc-qhp7] AWS API MCP 檔案存取限制繞過漏洞

原文來源：github.com/advisories
以下為核心要點摘要，附英文技術詞彙對照（紫色）幫助理解。
⚠️ 安全警告：如果你正在使用 awslabs.aws-api-mcp-server，請立即升級至 1.3.9 或更新版本。

漏洞概述

AWS API MCP Server（AWS API MCP 伺服器）存在 file access restriction bypass（檔案存取限制繞過）漏洞，攻擊者可透過 alternate paths（替代路徑）繞過 "no-access" 和 "workdir" 模式的保護，暴露本機任意檔案內容給 MCP client applications（MCP 客戶端應用程式）。

受影響資訊

套件名稱awslabs.aws-api-mcp-server (pip)
受影響版本≥ 0.2.14, < 1.3.9
修補版本1.3.9
CVE 編號CVE-2026-4270
CVSS 分數6.8（中等）
弱點類型CWE-424 — 替代路徑保護不當

漏洞原理

  • no-access 模式：設計上應完全阻止 AWS CLI（AWS 命令列工具）命令存取本機路徑，但可被繞過
  • workdir 模式：設計上應將操作限制在指定目錄內，但攻擊者可透過 path traversal（路徑遍歷）等手段存取其他位置
  • 影響：High confidentiality impact（高機密性影響），本機敏感檔案（如 SSH keys（SSH 金鑰）、credentials（憑證））可能被洩露

對程式設計師的意義

這個漏洞再次印證 MCP server 的安全邊界設計需要嚴格測試。如果你在 AI agent 工作流程中使用 AWS API MCP Server，攻擊者可能透過精心構造的工具呼叫讀取你機器上的任意檔案。立即執行 pip install awslabs.aws-api-mcp-server==1.3.9 升級，並審查你的 MCP server 是否都有做 input path sanitization（輸入路徑淨化）。

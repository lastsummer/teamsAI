Anthropic's Code Execution With MCP Cuts Agent Token Usage by 98.7%

原文來源：https://brightbean.xyz/blog/code-execution-mcp-efficient-ai-agents/
以下為核心要點摘要，附英文技術詞彙對照（紫色）幫助理解。

核心摘要

Anthropic 推出一種將 MCP 工具視為程式碼 API（code API）而非直接函式呼叫的模式，將 token（令牌）消耗從 150,000 降至 2,000。Agent 在沙盒環境（sandboxed environment）中寫入並執行程式碼，在本地處理資料而不將其暴露於模型的上下文窗口（context window）中。

98.7%
Token 消耗降低（150,000 → 2,000 tokens）

三大效率機制

  • 按需工具探索（on-demand tool discovery）：不需要在 context 中預載全部工具定義，避免上下文膨脹
  • 本地資料處理（local data processing）：中間資料在沙盒內過濾，防止大型資料集進入模型 context
  • 原生控制流（native control flow）：用標準程式設計結構（迴圈、條件判斷）取代多次模型呼叫

隱私與合規優勢

  • 敏感資料保留在沙盒環境內，不進入模型處理流程
  • 直接解決 GDPR（歐盟一般資料保護規則）和 HIPAA（美國醫療隱私法規）合規需求

技術需求與取捨

  • 需要沙盒基礎設施：安全沙盒、資源限制、執行監控
  • 依賴 LLM 的程式碼生成（code generation）能力
  • 成本和延遲的大幅改善值得這些架構投入

對程式設計師的意義

對於建立多工具 AI agent 的開發者，這個方法從根本上重構了效率經濟學。從 context 密集的工具定義轉移到程式碼式 API，讓處理數百個整合服務成為可能，而不會造成性能下降。98.7% 的 token 降低直接轉化為生產環境的成本節省，同時本地資料處理模式解決了之前阻礙在受監管行業中部署 agent 的企業合規限制。

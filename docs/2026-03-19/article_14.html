How to Build an MCP Client in Python in 10 Min

原文來源：https://dev.to/nebulagg/how-to-build-an-mcp-client-in-python-in-10-min-4k94
以下為核心要點摘要，附英文技術詞彙對照（紫色）幫助理解。

核心摘要

本文示範如何用 Python 建立 MCP client（MCP 客戶端），以程式化方式連接 MCP 伺服器，不依賴 Claude Desktop。透過 FastMCP（快速 MCP 框架）建立伺服器，使用 ClientSession（客戶端會話）進行工具探索和執行，並涵蓋 async/await（非同步等待）模式。

核心實作步驟

  • 建立 MCP 伺服器：用 FastMCP 透過 stdio transport（標準輸入輸出傳輸）暴露工具
  • StdioServerParameters：將伺服器作為子進程（subprocess）啟動，透過 stdin/stdout 通信
  • session.initialize()：執行 MCP 協議握手（handshake），必須在工具呼叫前執行
  • session.list_tools()：探索可用工具，包含名稱、描述和 schema（結構定義）
  • session.call_tool()：以參數呼叫特定工具並取得回應

程式碼模式

async with ClientSession(read, write) as session:
    await session.initialize()
    tools = await session.list_tools()
    result = await session.call_tool("tool_name", {"arg": "value"})

進階實作建議

  • 生產環境應支援 HTTP transport（HTTP 傳輸）而非僅限 stdio
  • 整合 LLM API（大型語言模型 API）讓 agent 自主選擇和呼叫工具
  • 加入 JSON-RPC（JSON 遠端程序呼叫）錯誤處理和 retry（重試）邏輯

對程式設計師的意義

建立自主 agent 或 Python 應用的開發者需要程式化的 MCP 工具存取，而不只是依賴 IDE 整合。這個模式啟用了執行期工具探索（runtime tool discovery）、動態 agent 決策，以及與語言模型的整合——這是生產 AI 系統管理多個工具來源的關鍵能力。AsyncIO（非同步輸入輸出函式庫）的使用模式也是現代 Python AI 開發的重要參考。

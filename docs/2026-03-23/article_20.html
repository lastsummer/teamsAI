Master Claude Code and Figma MCP for Design in 50 Minutes | Felix Lee

原文來源：creatoreconomy.so · 主講：Felix Lee
以下為核心要點摘要，附英文技術詞彙對照（紫色）幫助理解。

核心摘要

Felix Lee 曾訓練數千名設計師學習使用 Claude Code 和 Figma MCP（Model Context Protocol）。本集 podcast/文章展示如何將 Figma 設計稿直接轉換成可運作的網站，以及反向將網站匯回 Figma 可編輯圖層——消除傳統設計與開發之間的 handoff（交接）流程。

關鍵操作：設定 Figma MCP

三步驟設定
  1. 安裝：在 Claude Code 中執行 
    claude mcp add --transport http figma https://mcp.figma.com/mcp
  2. 驗證：輸入 /mcp，選擇 Figma，完成 OAuth 驗證
  3. 使用：複製 Figma frame 連結，貼上並輸入 Build this design

展示的三大應用場景

  • Figma → 網站：Live demo 15 分鐘內將設計稿轉成可運作的網頁
  • FigJam flowchart → 遊戲：將流程圖直接轉換成功能性遊戲
  • 3D 動態世界地圖：一次嘗試生成 3D animated world map（3D 動態世界地圖）

進階技巧

  • UX reviewer feedback（使用者體驗審查反饋）：讓 Claude 扮演 UX 評審角色提供改善建議
  • Code → Figma export：將生成的程式碼反向匯回 Figma 作為可編輯的設計稿
  • Landing page analysis（登陸頁面分析）：用 Claude 分析現有網站的設計品質

為何重要

Figma MCP 消除了設計與開發之間的傳統壁壘。設計師可以自行 ship（交付產品），不再依賴工程師實作；工程師則可以直接從設計稿生成 pixel-perfect（像素精確）程式碼，省去手動還原設計的時間。

對程式設計師的意義

Figma MCP 的三行設定指令值得現在就試試。不論你是否有設計背景，「複製 Figma 連結 → Build this design」的工作流程能大幅加速 UI 實作。特別是接外包或獨立開發者，掌握這個工具可以讓你一個人完成從設計到實作的完整流程。影片在 YouTube 上有 demo，可以直接觀看效果再決定是否採用。

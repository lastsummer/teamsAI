I Built a Self-Healing Production Server Using Claude Code — No API Key, No Extra Cost

原文來源：dev.to · 作者：Gumaro González
以下為核心要點摘要，附英文技術詞彙對照（紫色）幫助理解。

核心摘要

作者 Gumaro González 利用 Claude Code 建立了一個名為 PMO（Project Management Operations Agent（專案管理作業代理））的自主代理，部署於 Google Cloud Platform（GCP） 上的 Windows Server 2022，負責監控並自動修復六個 production services（生產服務）。

最關鍵的發現：claude -p（non-interactive flag（非互動模式旗標））直接使用現有的 Max 訂閱憑證執行，不需要獨立的 API Key，也不產生額外費用。

系統架構（三層設計）

Architecture
  1. Admin Interface（管理員介面）：透過 Telegram 以 !pmo 前綴發送指令
  2. Message Queue（訊息佇列）：使用 SQLite WAL DB 儲存任務佇列
  3. PMO Agent：每 10 秒輪詢一次，配備 24 個 MCP tools（MCP 工具），分六大類別

兩種運作模式

  • 直接指令模式：管理員透過 Telegram 下達命令，PMO 即時執行
  • 自主修復模式（Autonomous autocorrection（自主自動更正））：偵測到重複錯誤時自動觸發 → diagnose（診斷） → analyze（分析） → fix（修復） → verify（驗證） → report（回報） 的完整閉環

安全設計

  • Path traversal blocking（路徑穿越防護）：防止惡意路徑操作
  • Single concurrent execution（單一並發執行）：搭配 5 分鐘冷卻時間，避免重複觸發
  • System prompt constraints（系統提示約束）：限制 Claude 的操作範圍
  • CLI budget cap（CLI 預算上限）：每次執行最高 $2

實際成效（2026/03/21 起）

Results
  • 執行次數：9 次
  • 成功率：100%
  • 平均解決時間：~5 分鐘
  • 資料庫大小：84 KB

費用比較

  • 原始 API 模式：~$42/月（以每日 30 次執行計）
  • Max 訂閱方案：$100/月，但同時涵蓋互動式開發與其他自動化任務，實際 CP 值更高

對程式設計師的意義

這篇文章展示了一個重要的架構模式：用 claude -p 搭配 MCP tools 建立自主 agent loop（代理循環），不需要額外的 API 費用。這對於已有 Claude Max 訂閱的開發者來說，意味著可以免費將 Claude Code 嵌入 CI/CD pipeline（持續整合/部署流程） 或 自動化監控系統中。SQLite + Telegram 的輕量架構也值得借鑑，適合個人或小團隊的 self-healing infrastructure（自我修復基礎設施） 實踐。

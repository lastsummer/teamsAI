Set Up Claude Code Like Boris Cherny

原文來源：XDA Developers
以下為核心要點摘要，附英文技術詞彙對照（紫色）幫助理解。

核心摘要

Boris Cherny 是 Claude Code 的創作者（Anthropic 工程師），本文解析他本人如何使用 Claude Code，並提煉出可直接應用於個人工作流程的核心策略。

策略一：同時執行多個並行 Session

Cherny 最核心的使用模式是同時操作 10–15 個 concurrent Claude Code sessions（並行工作階段）：

  • 5 個在終端機（搭配 iTerm2 notifications（iTerm2 通知），完成時提醒）
  • 5–10 個在網頁瀏覽器
  • 手機上另開額外 session
實踐建議

作者實際測試後發現，光是同時跑 3–4 個 session 就能顯著改變工作流程。不需要達到 Cherny 的規模，平行處理本身就帶來效率提升。

策略二：優先使用 Plan Mode

Cherny 每次 session 都從 Plan Mode（計畫模式）開始，視策略規劃為必要步驟。Claude Code 在 Plan Mode 中會先詢問：

  • 需要整合哪些 integrations（整合項目）？
  • 使用什麼 tech stack（技術堆疊）？
  • 整體目標是什麼？
「A good plan is really important.」——Boris Cherny

策略三：保持「Vanilla」設定

Cherny 自描其設定「surprisingly vanilla（出乎意料地簡潔）」。Claude Code 刻意設計成可客製化、可 hack，但其創作者反而採用相對基本的配置，依然達到高效率。這說明工具本身已足夠強大，過度客製化未必必要。

注意事項

Cherny 在 Anthropic 工作，擁有大型工程團隊與完整的公司 codebase（程式碼庫）作為背景。但他的核心原則（Plan First、Parallel Sessions、保持簡潔）適用於任何規模的開發者。

對程式設計師的意義

從 Claude Code 的創作者本人學習使用方式，這是最直接的最佳實踐來源。最值得立即採用的是「Plan Mode 優先」——在開始任何較複雜的實作任務前先規劃，能顯著降低中途需要重新來過的機率。並行 session 策略則適合有多個獨立任務需要同時推進的開發者。

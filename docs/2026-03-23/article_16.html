Claude Code Secretly Hoards 140+ Config Files — Here's How to Take Control

原文來源：dev.to
以下為核心要點摘要，附英文技術詞彙對照（紫色）幫助理解。

核心問題

Claude Code 在使用者不知情的情況下，於 ~/.claude/ 目錄下累積了 140 個以上的設定檔，包含 memories（記憶）、skills（技能）、MCP server configs（MCP 伺服器設定）、hooks（鉤子）、plans（計畫）和 sessions（工作階段），散落在以 scope（範圍）為基礎的目錄中。

⚠️ 效能影響

Claude Code 每次啟動時會從目前 scope 及所有 parent scopes（父範圍）載入所有設定，導致 context window（上下文視窗）被無關設定佔用，降低 LLM 準確度。例如：放在 global scope（全域範圍）的 Python skill 會在每個 React 前端專案中也被載入。

具體問題清單

  • Scope contamination（範圍污染）：Python 技能出現在 React 專案中
  • Duplicate MCP entries（重複的 MCP 項目）：同一伺服器被初始化多次
  • Stale memories（過期的記憶）：已封存專案的設定仍消耗 tokens

解決方案：Claude Code Organizer

作者開發了一個開源管理工具，一行指令啟動：

npx @mcpware/claude-code-organizer

在 http://localhost:3847 提供視覺化儀表板，採用三欄介面：

  • Scope tree（範圍樹）：展示 Global → Workspace → Project 層級
  • Content area（內容區）：分類顯示所有設定項目
  • Detail panel（詳細面板）：檢視單一設定內容

八大管理類別

Memories、Skills、MCP Servers、Plans、Sessions、Config Files、Hooks、Plugins——支援跨 scope 拖曳移動、批次操作、即時搜尋，以及一鍵在 VS Code 中開啟。

技術架構

  • Scanner：檔案系統遍歷與解析
  • Mover：跨 scope 移轉（含驗證）
  • Server：REST API + 純 JS 前端
  • MCP Server mode：讓 Claude 能以程式化方式管理自身設定

對程式設計師的意義

如果你長期使用 Claude Code 並在多個專案中設定過 memories、MCP servers 或 skills，這個工具能幫你清理累積的設定廢棄物，讓每次 session 啟動時 context window 更乾淨。特別是有多個專案共用同一台機器的開發者，建議定期用此工具審查 global scope 中的設定，避免 scope contamination 悄悄降低回應品質。

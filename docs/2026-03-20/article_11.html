檔案智能中樞：搭載 MCP 的 AI 驅動企業盡職調查

原文來源：dev.to
以下為核心要點摘要，附英文技術詞彙對照（紫色）幫助理解。

核心摘要

這是 Notion MCP Challenge（Notion MCP 挑戰賽）的參賽作品，展示如何將兩個 MCP Server 協同工作，自動化企業盡職調查（corporate due diligence）流程。系統整合 Filed MCP Server（政府資料來源）與 Notion MCP Server（結構化輸出），讓一個 prompt 就能觸發完整的企業情報調查並自動整理成 Notion 工作區。

實測成果：以 $0.91 API 成本、37 次自主 AI 回合，調查三家國防承包商（Lockheed Martin、Anduril Industries、Palantir），發現 13 項可行動情報，包括 Anduril Industries 在美國 5 州以未登記外國公司身份運營 4-5 年。

系統架構：兩個 MCP Server 協同

Filed MCP Server（6 個工具，查詢政府資料）：

  • filed_search_entities — 跨 9 個美國州搜尋商業實體
  • filed_get_entity — 取得詳細實體資訊
  • filed_company_intel — 編排統一情報報告
  • filed_search_sec — 查詢 SEC EDGAR 申報資料
  • filed_search_contracts — 查詢 USASpending.gov 聯邦合約
  • filed_search_lobbying — 存取參議院 LDA 遊說資料庫

Notion MCP Server：將調查結果結構化為持久資料庫，包含類型化屬性（typed properties）與關聯關係（relationships）。

核心問題與解法

傳統合規研究需要手動跨多個分散的政府來源搜尋：州商業登記、SEC 申報、聯邦合約、遊說揭露。這個 MCP 架構把「跨資料源自動搜尋 + 結構化輸出」整合進單一 agent 對話流程。

安裝

npx filed-mcp-server
# Free tier: 100 次/月免費查詢，API key 申請：filed.dev/developers

對程式設計師的意義

這個案例展示了 MCP 多伺服器協同（multi-server orchestration）的強大潛力：不同的 MCP Server 各自負責不同資料源，AI agent 在中間協調，產生遠超單一 API 能力的複合結果。對於想建立企業資料整合工具或自動化研究流程的開發者，這是一個可直接參考的架構模式。

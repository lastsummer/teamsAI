Tonic Textual MCP Server：PII 遮蔽與 AI Agents 的結合

原文來源：securityboulevard.com
以下為核心要點摘要，附英文技術詞彙對照（紫色）幫助理解。

核心摘要

Tonic 發布了開源 MCP Server，將其 Textual 平台的 PII 去識別化（Personally Identifiable Information redaction，個人識別資訊遮蔽）功能整合進 AI agent 工作流程。開發者可透過 21 個 MCP 工具，在 agent 流程中自動偵測、轉換、遮蔽敏感個資，支援多種格式與語言。

21 個 MCP 工具涵蓋範圍

  • 文字處理：純文字、JSON、XML、HTML，支援 JSONPath（JSON 路徑表達式）精細控制遮蔽範圍
  • 檔案處理：PDF、Word 文件、試算表、圖片，保留原始格式；大檔案採後台處理，避免 agent 阻塞（blocking）
  • 批次操作：deidentify_folder 工具一次處理整個目錄，自動偵測檔案類型並平行處理
  • 資料集管理：建立資料集、上傳掃描、下載結果，全程透過 agent 指令完成

高度可設定的遮蔽選項

  • 支援 35+ 種實體類型（entity types）的個別設定
  • 多種合成模式（synthesis modes），包含長度感知（length-aware）選項
  • 跨文件的確定性替換映射（deterministic replacement mapping），同一實體在不同文件中替換結果一致
  • 支援自訂實體模型（custom entity models）與允許/封鎖清單

典型使用場景

  • 準備 AI 訓練資料集（去除個資後才能安全使用）
  • 建立合規的 RAG pipeline（檢索增強生成管線）
  • 自動化文件審查工作流程
  • 處理資料匯出時的隱私保護

技術架構

採用 HTTP streaming transport（HTTP 串流傳輸），預設並發 API 呼叫上限 50 次，透過 MCP task API（MCP 任務 API）實作後台任務處理，並內建暫時性故障的自動重試邏輯（retry logic）。

對程式設計師的意義

這是目前少數將資料隱私合規（data privacy compliance）與 AI agent 工作流程直接整合的開源工具。對於需要處理真實用戶資料的 RAG 系統或 AI 應用開發者，這個 MCP Server 提供了一條合規路徑，而不需要自行實作複雜的 PII 偵測邏輯。

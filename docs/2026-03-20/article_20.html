MCP Server 教學：逐步建立你的第一個 Model Context Protocol Server

原文來源：dev.to
以下為核心要點摘要，附英文技術詞彙對照（紫色）幫助理解。

核心摘要

這是一篇完整的 MCP Server 實作教學，使用 TypeScript（TypeScript）與官方 Anthropic SDK（Anthropic 軟體開發套件），從零開始建立你的第一個 Model Context Protocol Server（模型上下文協議伺服器）。

MCP 是 Anthropic 建立的開放標準，定義 AI 助手與外部系統的通訊方式，作用如同通用連接器（universal connector），讓 AI 客戶端無需自訂整合程式碼即可使用外部工具。

三個核心原語（Primitives）

  • Tools（工具）：AI 模型可主動呼叫的函式，用於執行操作，由模型控制
  • Resources（資源）：透過 URI（統一資源識別碼）存取的唯讀資料，由應用程式控制
  • Prompts（提示）：可重複使用的模板，引導 AI 行為，由使用者控制

建立步驟

  1. 專案初始化：建立 Node.js（Node.js）專案，安裝 @modelcontextprotocol/sdk 與 Zod（用於輸入驗證的 schema 庫）
  2. 建立 Tools：使用 server.tool() 註冊可呼叫工具，提供足夠精確的描述讓 AI 模型能正確選用。範例：UUID 產生、JSON 格式化、Token 計數、JSON Schema 驗證
  3. 加入 Resources：透過 URI 端點暴露靜態或動態資料，例如伺服器狀態資源、文件模板系統
  4. 實作 Prompts：建立結構化引導模板，例如包含特定審查標準的程式碼審查 prompt
  5. 設定傳輸層：選擇 stdio（標準輸入輸出，適合本地開發）或 SSE（Server-Sent Events，適合 HTTP 遠端存取）

生產環境最佳實踐

  • 工具描述要具體、精準，這直接影響 AI 的選用決策
  • 使用嚴格 schema（結構描述）驗證所有輸入
  • 日誌寫到 stderr（標準錯誤輸出），因為 stdout（標準輸出）保留給協議訊息
  • 為外部呼叫實作 timeout（逾時處理）
  • 遵守最小權限原則（principle of least privilege）

連接 Claude Desktop

// Claude Desktop 的 MCP 設定（macOS）
// ~/Library/Application Support/Claude/claude_desktop_config.json
{
  "mcpServers": {
    "my-server": {
      "command": "node",
      "args": ["/path/to/your/server.js"]
    }
  }
}

測試與除錯

官方提供 MCP Inspector（MCP 視覺化偵錯工具）：

npx @modelcontextprotocol/inspector

對程式設計師的意義

這篇文章是 MCP Server 開發的完整入門路徑，適合想把自己的工具或 API 接入 Claude、Kiro 等 AI 客戶端的開發者。掌握這三個原語（Tools/Resources/Prompts）之後，就能系統性地將現有服務封裝成 MCP Server，解鎖與各種 AI 工具的互通性。

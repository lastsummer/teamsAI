MCP Server of You：為 Kiro IDE 建立個人知識力量

原文來源：dev.to · 作者：Michael Rewiri-Thorsen · 2026-03-20
以下為核心要點摘要，附英文技術詞彙對照（紫色）幫助理解。

核心摘要

作者開發了 kiro-recall，一個能把個人知識庫自動注入 Kiro IDE 工作階段的 MCP Server（模型上下文協議伺服器）。核心概念是「第二大腦」（second brain）：在你輸入第一個 prompt 之前，系統已主動將你的筆記載入模型上下文，解決工程師因知識積累而導致的認知過載問題。

知識庫以 Obsidian vault（Obsidian 知識庫）形式存放於本地端，採用 Zettelkasten 方法論（卡片盒筆記法），純 Markdown 檔案，無雲端依賴。

架構設計：七層目錄 + 四個 Hook

Vault 採七層目錄結構，最關鍵的是 06-permanent/（每次工作階段都會載入的永久筆記）與 00-inbox/（當下捕捉的暫存筆記）。

系統透過四個 Hook（鉤子/觸發器）驅動：

Hook觸發時機功能
Session Start每次工作階段開始自動讀取並注入 Vault 上下文
/recall使用者手動觸發將當前靈感/發現存入 inbox
Vault Scaffold初次設定建立七層目錄結構
/promote使用者手動觸發將 inbox 筆記升格為永久原子筆記

關鍵要點

  • 四階段相關性偵測（relevance detection）：依工作區資料夾名稱→開啟檔案路徑→全部永久筆記→空 Vault 引導，自動決定載入哪些筆記
  • Hook 實作方式：.kiro.hook JSON 檔，包含觸發類型與 agent 指令，底層用 MCP tool call（MCP 工具呼叫）讀取 Markdown 檔案
  • /promote 流程：將 inbox 筆記重新格式化為原子筆記（atomic notes），包含「想法」「重要性」「相關連結」「來源元資料」四個區塊，原始檔案自動刪除
  • 無編譯程式碼：整個架構不需要任何 compiled code（已編譯程式碼），每個 Hook 本質上只是一個包含 prompt 的 JSON 檔

安裝方式

# 透過 Kiro IDE 的「Add Power from GitHub」安裝
# https://github.com/mikeartee/kiro-recall

# 在 mcp.json 設定 vault 路徑後，重新啟動 Kiro 即可

對程式設計師的意義

這篇文章展示了一個高度實用的 MCP Server 落地案例：用最小化架構（純 JSON + Markdown）打造個人知識與 AI 工作流程的深度整合。對於使用 Kiro IDE 或有意打造自訂 MCP Hook 的開發者，kiro-recall 是一個值得參考的完整實作範例。作者正在尋找 Windows/Mac 使用者測試，是參與早期回饋的好機會。

Diffity — GitHub-style diff viewer for AI coding tools（GitHub 風格 Diff 檢視器，專為 AI 程式碼工具設計）

原文來源：github.com/kamranahmedse/diffity
以下為核心要點摘要，附英文技術詞彙對照（紫色）幫助理解。

核心摘要

Diffity 是一個 agent-agnostic（與 AI 代理無關，通用型） 的程式碼審查平台，提供類似 GitHub 的 diff 檢視介面（diff viewer interface）。它可以與 Claude Code、Cursor、Codex 等 AI 程式設計助理無縫整合，讓開發者在 AI 生成的程式碼提交前進行可視化的程式碼審查（code review）。

核心功能

  • Diff 檢視：以語法高亮（syntax highlighting） 比較 working tree（工作樹）、commit（提交）、branch（分支）、tag（標籤） 之間的差異
  • AI 程式碼審查：AI agents 可分析 diff 並留下帶有嚴重性標籤（severity tags） 的行內評論（inline comments）
  • File Browser（檔案瀏覽器）：以語法高亮探索整個 repo，並支援評論功能
  • Guided Tours（導覽功能）：自動生成逐步的程式碼庫導覽，含高亮說明
  • GitHub PR 整合：將 PR 拉取到本地審查，並將評論同步回 GitHub
  • Multi-Project 支援：同時在自動分配的 port 上運行多個實例

安裝與使用

# 全域安裝
npm install -g diffity

# 整合到 AI agent skill
npx skills add kamranahmedse/diffity

安裝後可使用以下指令：

diffity          # 審查未提交的變更（uncommitted changes）
diffity HEAD~1   # 審查最後一次 commit
diffity main     # 與 main branch 比較
diffity v1.0.0 v2.0.0  # 比較兩個 release 版本

在 Claude Code 中使用的 Slash Commands

  • /diffity-diff — 開啟 diff 檢視器
  • /diffity-review — AI 自動審查並留下評論
  • /diffity-resolve — 套用 AI 或手動審查建議
  • /diffity-tree — 瀏覽完整專案結構
  • /diffity-tour — 生成程式碼庫導覽

對程式設計師的意義

在 vibe coding（氛圍程式設計） 盛行的現在，AI 生成的程式碼量大幅增加，但「AI 生成 ≠ 正確且安全」。Diffity 提供了一個可視化的審查閘門（review gate），讓開發者在 AI 的速度和人類的品質控制之間取得平衡。特別適合在 Claude Code、Cursor 等工具中，於 commit 前進行快速的目視審查，是減少 AI 生成程式碼引入 bug 的實用工具。

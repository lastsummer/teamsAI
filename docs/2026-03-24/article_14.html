I Migrated a Laravel App to a Custom PHP Framework in One Claude Code Session（我在一個 Claude Code 會話中將 Laravel 應用程式遷移到自訂 PHP Framework）

作者：Russell Jones｜2026-03-23｜原文來源：dev.to
以下為核心要點摘要，附英文技術詞彙對照（紫色）幫助理解。

核心摘要

Russell Jones 記錄了一次驚人的工程壯舉：在約 8 小時的單一 Claude Code session 中，將 GoFormX（一個表單管理平台（forms management platform））從 Laravel 12 完整遷移到 Waaseyaa（一個自訂 PHP framework（框架））。這不只是遷移，過程中還從零發布了三個 Packagist 套件（packages）。

成果一覽

  • waaseyaa/inertia — 伺服器端 Inertia v3 adapter（適配器），含 30 個測試
  • waaseyaa/auth — Headless 認證系統（headless authentication），含 46 個測試
  • waaseyaa/billing — Stripe 付款整合（Stripe integration），含 35 個測試
  • 完整應用：7 個 controllers（控制器）、37 個 unit tests（單元測試）、28 個 Vue 3 components（元件）及完整基礎設施

TDD Pipeline 方法論

Jones 採用可重複的 TDD pipeline（測試驅動開發流水線），每個步驟都有具體可驗證的結果：

  1. Scaffolding（鷹架搭建） — 建立基本結構
  2. Test-driven development（測試驅動開發） — 先寫測試再實作
  3. Wiring components（連接元件） — 整合各模組
  4. Verification（驗證） — 確認功能正常
  5. Deployment（部署） — 上線

Jones 強調：「Claude Code 能可靠執行這個 pipeline，因為每個步驟都有具體且可驗證的結果。」

技術挑戰與解決方案

  • 資料庫相容性：Waaseyaa 使用 SQLite kernel 但 GoFormX 需要 MariaDB — 透過自訂 UserRepository（使用者儲存庫） 解決
  • Inertia v3 page data format（頁面資料格式） — 需要直接閱讀原始碼才能發現差異
  • PHP-FPM 環境變數繼承（environment variable inheritance） — 透過設定調整解決

端對端測試方式

Jones 不使用基本的 curl 請求，而是採用 Playwright MCP（瀏覽器自動化 MCP 伺服器） 進行端對端測試（end-to-end testing），驗證真實瀏覽器行為。

對程式設計師的意義

  • 可重複的 Pipeline：這個案例證明，只要每個步驟有明確的可驗證結果，Claude Code 就能作為可靠的工程執行引擎。
  • 從框架遷移的新可能：過去需要數週的框架遷移，現在可以在一個工作天內完成。
  • TDD 與 AI 的協同：TDD 的測試先行特性，使 AI 有清晰的「完成標準」，大幅提升遷移可靠性。
  • MCP 工具鏈：Playwright MCP 搭配 Claude Code，讓 AI 能夠自主完成完整的 E2E 測試驗證。
